# Lessons Learned

- Added Agent Communication Rules and Mandatory End-of-Task Behavior to `.agents/AGENTS.md`. Discovered that we should not be conversational and must keep track of lessons learned.
- Discovered that removing OSSF Scorecard involves removing the `.github/workflows/scorecard.yml` file, the badge in `README.md`, and closing any related automated issues.
- Discovered that if `gh pr merge` fails on a Dependabot PR, it is likely due to merge conflicts or branch protection rules rather than OAuth scope limitations, as the `gh` CLI does have access to merge workflows. Always check for and resolve conflicts first.
- Discovered that when using absolute paths, the username in the home directory path must be carefully checked (e.g. `tengischinzorigt`) to prevent permission denied errors during directories creation.
- Discovered that when writing unit tests for CLI tools using Cobra in a monorepo, mocking the root environment variable and pre-seeding target directories inside a `t.TempDir()` allows full command validation and scaffolding testing without causing side-effects or requiring interactive prompts.
- Discovered that unit/infrastructure testing of Terraform plans can be performed offline and without AWS credentials by starting a local Go `httptest.NewServer` to mock STS, EC2, and EKS endpoints, writing a `providers_override.tf` redirecting endpoints to it, and deleting `backend.tf` to fallback to a local state backend.
- Discovered that when writing E2E validation tests for a GitOps developer CLI tool (`pave`), compiling the binary locally into a temporary test folder and executing it programmatically with `PAVESTACK_ROOT` pointing to the repository allows verifying the full CLI behavior including file generation and schema-based parameter validations, followed by a deferred cleanup of generated files.
- Discovered that verifying existing React Vitest unit tests in `pavestack-portal` can be performed cleanly by using `npx vitest run` or the `npm run test` script, and is fully integrated into the monorepo's `make test` pipeline, validating both logic helpers and UI components rendering using `@testing-library/react` and `jsdom` environments.
- Discovered that coordinating a multi-milestone testing strategy using parallel worker subagents (Go, Frontend, Infra, E2E GitOps) keeps task context clean and execution efficient, and network/stream failures can be recovered seamlessly using the Replace pattern to spawn a generation 2 successor worker.
- Discovered that performing a comprehensive victory audit involves verifying file modification patterns, conducting code audits to ensure no facade patterns or hardcoded assertion results exist, and independently running all tests across monorepo components (Go, Vite/React, Terraform, and compiled Go binary E2E tests) using local workspace runners.
- Discovered that custom skills located in non-standard paths like `~/.agents/skills` can be registered globally or workspace-wide in Gemini by defining them in a `skills.json` config file or by symlinking the directory to a standard customization root.
- Discovered that direct agent write operations outside the workspace via the API are blocked by system boundaries, but configuration changes can be completed successfully using terminal execution redirects with explicit user approval.
- Discovered that when launching the teamwork multi-agent system, it is crucial to spend time iteratively crafting the prompt with the user (eliciting ambiguity, setting integrity modes, and designing objective verification criteria) before delegating, as this prevents premature self-certification by the agents.
- Discovered that refactoring template generation in Go from fmt.Sprintf with positional parameters to text/template improves readability and maintainability of multi-line Kubernetes/ArgoCD manifests, and encapsulating git/gh shell command execution into a separate module simplifies testing by isolating tool validation checks.
- Discovered that performing forensic audits in benchmark integrity mode requires verifying that there are no third-party package dependencies used for core features, that Go standard `text/template` library is correctly used for manifest rendering, and that code modularization (e.g. separating manifest rendering from git/gh operations) is properly structured without facades or hardcoded checks.
- Discovered that refactoring and modularizing a React codebase by extracting inline SVGs and sub-components into separate files (`icons.tsx`, `components.tsx`, `App.tsx`, and `index.ts`) improves code organization and readability, and importing them correctly in both `main.tsx` and `main.test.tsx` ensures that Vitest and DOM testing environments continue to pass successfully without regressions.
- Discovered that when auditing React component modularization (monolith extraction) under benchmark integrity mode, ensuring absolute separation of icons (`icons.tsx`), structural components (`components.tsx`), core layout (`App.tsx`), and entry point exports (`index.ts`) allows verifying code quality and structure without violating layout guidelines, while preserving test suite passability.
- Discovered that when refactoring file system operations to use the `afero.Fs` interface, since `afero` does not export `WalkDir` in version `v1.15.0`, implementing a custom wrapper or helper `WalkDir` that uses `afero.Walk` under the hood allows maintaining the standard library `fs.WalkDirFunc` signature. Refactoring tests to use `afero.NewMemMapFs()` ensures they run fully in-memory without accessing or writing to the physical disk.
