What Pavestack does

The paved road from idea to running service.

01

Self-service scaffolding

pave create-service --name payments --team team-payments clones the golden-path Go API from service-template-api, renders the GitOps manifests into platform-config/tenants/payments/, and opens the pull request — one command, no scaffolding boilerplate written by hand, no waiting on a platform engineer to hand-roll it. 

services/payments-api/
  cmd/, internal/, Dockerfile, otel.go

platform-config/tenants/payments/
  ├─ application-set.yaml
  ├─ namespace.yaml
  ├─ network-policy.yaml
  ├─ resource-quota.yaml
  └─ kyverno-labels.yaml
02

GitOps delivery, not console access

CI builds the image, signs it with Cosign, and generates an SBOM with Syft before it ever reaches ECR. Once the PR merges, an Argo CD ApplicationSet — one per tenant, never a hand-maintained Application — reconciles the cluster to match git. Nobody runs kubectl apply.

03

Policy & security by default

Every tenant namespace ships default-deny by NetworkPolicy, resource-quota'd, and Kyverno-policed — no privileged containers, no :latest tags, mandatory labels and limits enforced at admission, not in a review checklist. Cost-tagged for FinOps attribution from day one.

Golden path

One command in, one running service out.

Every service takes the identical path from a developer's terminal to a workload running in EKS — no bespoke exceptions.

Architecture

Layers, not a monolith.

Platform infrastructure is provisioned once. GitOps config reconciles many tenants on top of it. Observability and policy wrap every layer instead of bolting onto one.

Why GitOps

The cluster only ever matches what's in git.

Not a philosophy — a concrete difference in who can touch production and how you find out what happened.

Before kubectl apply, from a laptop

  • Whoever has the kubeconfig can change production, right now, unreviewed.
  • "What changed?" means grepping shell history and Slack threads.
  • Drift between what's running and what's documented is discovered by incident.
  • Rollback means someone remembers the previous flags and re-runs the command by hand.

After PR merges, Argo CD reconciles

  • No standing cluster credentials to leak, share, or rotate — access is a pull request, reviewed like code.
  • Every change to production has an author, a diff, and an approval in git history.
  • Argo CD continuously diffs live state against git and self-heals drift automatically.
  • Rollback is git revert — Argo CD reconciles the cluster back, no tribal knowledge required.

Open source

Apache-2.0. Built in the open. Run it on your own cluster.

Pavestack is not a hosted product with a proprietary control plane — it's a monorepo you fork, adapt to your org's Terraform conventions, and operate yourself. Issues and pull requests are welcome, especially around new golden-path templates and additional Argo Rollouts analysis providers.
